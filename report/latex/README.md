# Report (NeurIPS 2025 style) — how to build & submit

This folder is the **submittable report** for the "Project report (60%)" deliverable.
It compiles to a 15-page PDF in the required NeurIPS 2025 LaTeX style.

```
report/latex/
├── main.tex          # the report source (edit here)
├── main.pdf          # compiled output  <-- this is what you submit
├── references.bib    # bibliography
├── neurips_2025.sty  # official NeurIPS 2025 style (do not edit)
└── figures/          # all figures used by the report (generated by experiments/)
```

## What to submit
Submit **`main.pdf`** on the class Slack channel (deadline: **July 18**).
The rubric it targets:

| Rubric item | Where in the report |
|---|---|
| Motivation & data/task (9) | §1 Introduction, §2 Problem Formulation & Dataset |
| Model appropriateness (9)  | §3 Background, §4 Models (incl. proposed model) |
| Insights + results (9)     | §7 Results, §8 Ablation, §9 Discussion |
| Figures (9)                | Figures 1–8 (curves, bars, confusion matrix, scatter) |
| Code snippets (4)          | Listings 1–5 (model, data, training loop, config, runner) |

## Build

### Option A — Overleaf (easiest, nothing to install)
1. Zip this `latex/` folder (or upload the files).
2. New Project → Upload Project → select the zip.
3. Set the main document to `main.tex`, compiler **pdfLaTeX**, then Recompile.

### Option B — Local with Tectonic (self-contained, recommended)
```bash
brew install tectonic     # macOS; see tectonic-typesetting.github.io for others
cd report/latex
tectonic main.tex         # produces main.pdf
```

### Option C — Local with a full TeX distribution
```bash
cd report/latex
pdflatex main
bibtex main
pdflatex main
pdflatex main
```

## Updating figures
The figures are copied from the repo's top-level `figures/` folder, which is
regenerated by the experiment scripts (`experiments/run_*.py`). After re-running an
experiment, refresh them with:
```bash
cp ../../figures/*.png figures/
```
Then rebuild. All numbers in the tables come from `results/*.csv`.
